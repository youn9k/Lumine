iOS 코드 작성 AI 에이전트 규칙

언어 및 커뮤니케이션

- 모든 응답과 문서는 한국어로 작성한다
- 코드 주석도 한국어로 작성한다
- 변수명, 함수명 등 코드 자체는 영어를 사용하되, 설명은 한국어로 제공한다

유지보수성 원칙

- 명확한 네이밍: 변수, 함수, 클래스명은 역할과 의도를 명확히 드러내도록 작성한다
- 단일 책임 원칙(SRP): 각 클래스와 함수는 하나의 명확한 책임만 가지도록 설계한다
- 가독성 우선: 짧은 코드보다 읽기 쉬운 코드를 우선한다
- 매직 넘버/문자열 제거: 하드코딩된 값은 상수로 정의한다
- 적절한 주석: 왜(Why)를 설명하는 주석을 작성한다 (무엇을 하는지는 코드로 표현)

Swift 코딩 규칙

- Swift API Design Guidelines를 준수한다
- 옵셔널은 적절히 처리하며, force unwrap(!)은 정당한 이유가 있을 때만 사용한다
- 에러 처리는 명시적으로 한다 (try?, try! 남용 지양)
- 강한 참조 순환을 방지한다 ([weak self]를 지향)

코드 작성 시 고려사항

- 확장 가능성을 고려한다
- 테스트 가능한 코드를 작성한다
